Apuntes de PHP y MySQL (Basado en Ejercicios)

Análisis de funciones y lógica de los 14 archivos proporcionados.

1. Configuración y Conexión

Archivo: config3.php
<?php $SERVIDOR = "localhost"; $USUARIO = "paco"; $CLAVE = "alumno"; $DATABASE = "jardineria"; ?>
Función: Almacenar las credenciales en variables.
  • Este archivo no tiene lógica, solo definición de variables.
  • Se usa posteriormente con include() para no repetir datos.

2. Inserción de Datos (Alta)

Archivos: insertarOficina.html → insertar3.php

El flujo comienza con un formulario HTML que envía datos por POST.

if (isset($_POST['INSERTAR'])){ include ("config3.php"); $cnx = mysqli_connect($SERVIDOR,$USUARIO,$CLAVE,$DATABASE) or die(mysqli_error($cnx)); // Construcción manual de valores SQL $valores = "'".$_POST['codOfi']."',"; $valores .= "'".$_POST['ciudadOfi']."',"; // ... (concatenación del resto de campos) $sql = "INSERT INTO Oficinas VALUES($valores)"; $res = mysqli_query($cnx,$sql) or die(mysqli_error($cnx)); }
Funciones Clave:
  • isset($_POST['...']): Verifica si se ha pulsado el botón del formulario.
  • include("config3.php"): Carga las variables definidas en el paso 1.
  • mysqli_connect(...): Abre la conexión al servidor MySQL.
  • or die(mysqli_error($cnx)): Si falla la instrucción anterior, detiene la ejecución y muestra el error exacto que devuelve MySQL.
  • mysqli_query($cnx, $sql): Envía la sentencia SQL a la base de datos.

3. Listados (Lectura)

Archivos: listado2.php, listado3.php

Estos archivos muestran cómo recuperar datos. Se utilizan dos métodos principales con mysqli_fetch_array.

Método A: Usando list()

while (list($codOfi,$ciudadOfi) = mysqli_fetch_array($resultado)){ echo "CODIGO: ".$codOfi."<br>"; }
Explicación:
  • mysqli_fetch_array($resultado): Obtiene una fila de resultados como un array (numérico y asociativo mezclado).
  • list($var1, $var2): Asigna las columnas devueltas (en orden 0, 1, 2...) a variables individuales (`$codOfi`, `$ciudadOfi`).
  • Es el método preferido en listado3.php.

Método B: Sin list() (Array directo)

while ($ciudades = mysqli_fetch_array($resultado)){ echo "CODIGO: ".$ciudades['codigoOficina']."<br>"; }
Explicación:
  • Aquí se accede al dato usando el nombre de la columna entre corchetes, ej: $ciudades['codigoOficina'].

4. Búsqueda y Paso de Parámetros

Archivos: buscarOficina.html, buscarPais3.php, ver3.php

Se utilizan enlaces o formularios GET para pasar información en la URL (ej: ver3.php?codOficina=MAD-ES).

// En buscarPais3.php (Búsqueda por patrón) $consulta = "SELECT ... FROM Oficinas WHERE pais LIKE '%".$_GET['paisOfi']."%'"; // En ver3.php (Búsqueda exacta por PK) $consulta = "SELECT * FROM Oficinas WHERE codigoOficina='".$_GET['codOficina']."'";
Funciones y Lógica:
  • $_GET['...']: Recoge el dato enviado por la URL o formulario GET.
  • LIKE '%...%': Operador SQL para buscar coincidencias parciales (usado en buscarPais3.php).
  • mysqli_num_rows($resultado): Cuenta cuántas filas devolvió la consulta. Se usa para saber si "Existe la oficina" o "No se obtuvieron resultados".
  • mysqli_free_result($resultado): Libera la memoria ocupada por los datos obtenidos (muy importante y usada en todos los listados).

5. Eliminación

Archivos: eliminarOficina.html → eliminar3.php
if (isset($_GET['codOficina'])){ // ... Conexión ... $sentencia = "DELETE FROM Oficinas WHERE codigoOficina='".$_GET['codOficina']."'"; $resultado = mysqli_query($cnx,$sentencia) or die(mysqli_error($cnx)); echo "Oficina eliminada: ". $_GET['codOficina']; }
Flujo:
  • Recibe el código por GET (desde el enlace en el listado o el formulario).
  • Ejecuta la sentencia SQL DELETE.
  • Cierra la conexión con mysqli_close($cnx).

6. Edición (Modificación)

Archivos: editar3.html → editar3.php → actualizar3.php

Este es el proceso más complejo, dividido en 3 pasos (como se ve en tu Gráfico JPG).

Paso 1: Seleccionar (editar3.php)

Recibe el código, hace un SELECT y rellena un formulario HTML con los datos actuales.

$consulta = "SELECT * FROM Oficinas WHERE codigoOficina='".$_GET['codOfi']."'"; $resultado = mysqli_query($cnx,$consulta)... $ofi = mysqli_fetch_array($resultado); echo "<form action='actualizar3.php' method='POST'>"; echo "CODIGO: <input type='text' name='codigoOfi' value='".$ofi['CodigoOficina']."' readonly/>";

Paso 2: Actualizar (actualizar3.php)

Recibe los datos modificados por POST y ejecuta el UPDATE.

$valores = "CodigoOficina='".$_POST['codigoOfi']."',"; $valores .= "Ciudad='".$_POST['ciudadOfi']."',"; // ... concatenación ... $sql = "UPDATE Oficinas SET $valores WHERE CodigoOficina='".$_POST['codigoOfi']."'";
Detalles:
  • El campo 'CodigoOficina' se muestra como readonly en el formulario para evitar que el usuario cambie la clave primaria.
  • En actualizar3.php, se construye la string $valores concatenando campo a campo.
  • Se usa la cláusula WHERE para asegurar que solo se modifica esa oficina específica.
